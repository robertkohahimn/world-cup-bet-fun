## Domain rules (locked — resolves spec ambiguities)

1. **Line** = (match, favored team, spread S ≥ 0 in 0.5 steps), set per-room by the room admin. "Mexico +1" means Mexico must win by **margin > 1** for Mexico-backers to win; margin == S → **push** (nobody wins); margin < S (incl. draw/loss) → other side wins.
2. **Betting window:** opens when the line is set, closes at `kickoff − 6h`. Bets are upsertable until the deadline. Admin may set/adjust the line only until the same deadline.
3. **Settlement** (per room, per match with a line, runs once when the final score is recorded):
   - *Participants* = members whose `joined_at` < betting deadline (late joiners are exempt).
   - *Winners* = participants who bet the winning side. *Losers* = participants who bet the losing side **plus participants with no bet** (spec: no bet counts as a loss).
   - Each winner gets `+L` points where `L` = number of losers; each loser gets `−1`. (Spec's example: 1 winner, 3 losers → +3 / −1×3.)
   - **Push:** bettors get 0; non-bettors still count as losers (−1) but there are no winners to collect.
4. **Visibility:** members' picks are hidden from each other until the deadline passes (prevents copying); after deadline all picks are visible.
5. **Results** are global (one final score per match) and entered by a **platform admin** (first registered user; `is_admin` flag). Recording a score settles every room that has a line on that match, idempotently.
6. Room points start at 0 and may go negative. Leaderboard ties broken by wins, then earlier join.

`side='cover'` = the favored team covers the spread; `'against'` = it doesn't.

## Settlement engine (complete spec)

```ts
type LineOutcome = 'cover' | 'against' | 'push';
export function evaluateLine(favoredGoals: number, otherGoals: number, spread: number): LineOutcome {
  const margin = favoredGoals - otherGoals;
  if (margin > spread) return 'cover';
  if (margin === spread) return 'push';
  return 'against';
}
export interface Participant { userId: number; side: 'cover' | 'against' | null }
export function settleRoomMatch(outcome: LineOutcome, ps: Participant[]): {userId:number; delta:number; result:'win'|'loss'|'push'}[] {
  const losers  = ps.filter(p => p.side === null || (outcome !== 'push' && p.side !== outcome));
  const winners = outcome === 'push' ? [] : ps.filter(p => p.side === outcome);
  return ps.map(p => losers.includes(p) ? {userId:p.userId, delta:-1, result:'loss'}
    : winners.includes(p) ? {userId:p.userId, delta:losers.length, result:'win'}
    : {userId:p.userId, delta:0, result:'push'});
}
```

Test cases (vitest): spec example (1v3 → +3/−1×3); push with integer spread; push with non-bettor (−1, no winners); all same winning side (+0 each); all losers; non-bettor as sole loser (+1 winners); 0-spread draw → push; against-side wins on draw with spread 1.
